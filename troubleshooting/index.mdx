Log errors, dead ends, and solutions here so you don't solve the same problem twice.

## How to add a page

1. Create an MDX file — e.g. `troubleshooting/isaac/isaac-lab.mdx`.
2. Register it in `docs.json` → **Troubleshooting** tab → under the right group `pages` array.
3. Run `mint dev` locally to preview. Push to deploy.

Each topic is one long page. Add a new `###` section at the top for each issue — newest first.

## How to add an image

Put screenshots under `troubleshooting/isaac/images/<topic>/` and reference them with an absolute path:

```mdx
<Frame caption="What this screenshot shows">
  <img src="/troubleshooting/isaac/images/isaac-sim/gpu-error.png" alt="GPU driver error dialog" />
</Frame>
```

Use PNG or WebP. Name files by issue slug (e.g. `conda-env-mismatch.png`) so they stay easy to find.

## Issue template

```markdown
### [Short title] — YYYY-MM-DD

**Environment:** OS, versions, tools involved

**Problem:** What went wrong and any error messages.

**Cause:** Why it happened (if known).

**Solution:** Steps that fixed it.

**Links:** Related docs or threads.
```
